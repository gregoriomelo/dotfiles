# Documentation
- Propose an ADR (Architecture Decision Record) in `docs/adr/` BEFORE implementing any change that introduces a new dependency, language, or core system tool.
- Maintain relevant documentation.
- When creating documentation, add reference links in existing docs to the new docs. Always cross reference files.
- Avoid duplicating docs.
- When completing a task, make sure all documentation is up to date.
- When completing a task, use the `@task-recorder` skill to generate a summary of the work.
- Include the task documentation in the final commit of the task.
