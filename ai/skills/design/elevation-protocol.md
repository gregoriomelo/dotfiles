# Elevation Protocol

A systematic process for taking any visual output from functional to polished. Run through this in sequence. Never skip a phase.

---

## Phase 1: Functional Skeleton

**Goal:** Make it work before making it beautiful.

Build the complete information architecture:
- All required content is present
- Logical hierarchy is established (what matters most, second, third)
- Layout makes structural sense
- Nothing is missing

**Checkpoint:** If all styling were removed, would this communicate its purpose clearly?

---

## Phase 2: Typography Audit

**Goal:** Establish a type system that creates hierarchy without requiring color.

Actions:
1. Define the type scale: 3–5 size steps maximum. Name them (display, heading, body, label, caption).
2. Assign weights purposefully: 700+ for primary headings, 500–600 for secondary, 400 for body.
3. Set line heights for each level. Headlines tight. Body generous.
4. Check tracking: add letter-spacing to all-caps labels (0.08em minimum).
5. Verify measure on all body text: 45–75 characters per line.
6. Apply tabular numerals to all data contexts.

**Checkpoint:** Can you determine the information hierarchy by squinting at the page? If not, the type system isn't working.

---

## Phase 3: Color Refinement

**Goal:** Apply color purposefully, not decoratively.

Actions:
1. Start with full desaturation — build the layout in grayscale.
2. Identify the single most important action or piece of information.
3. Apply the accent color only there.
4. Replace pure black with near-black, pure white with off-white.
5. Build neutral range: at least 5 gray steps between background and text.
6. Check all text contrast ratios (WCAG AA minimum).
7. Ask: "Does removing any color element hurt communication?" If no, remove it.

**Checkpoint:** Could this work without color and still be clear? Color should enhance, not be the only signal.

---

## Phase 4: Spacing & Rhythm

**Goal:** Create consistent spatial rhythm that makes scanning feel effortless.

Actions:
1. Define the base unit (4px or 8px).
2. Audit all internal component padding — should be at least 2× base unit.
3. Audit section spacing — should be at least 4–6× base unit between major sections.
4. Look for crowded areas and add space generously. Space is free; use it.
5. Look for orphaned elements — things that float without clear spatial relationship to others.
6. Apply consistent vertical rhythm to text blocks.

**Checkpoint:** Does the layout breathe? Is there a consistent spatial grammar across the whole piece?

---

## Phase 5: Detail Pass

**Goal:** Address the micro-level craft details that separate polished from generic.

Actions:
1. Check optical alignments — do things look aligned, not just measure as aligned?
2. Verify icon/text optical sizing — icons should appear the same visual size as adjacent text.
3. Audit border radii — are they consistent? Do they match the character of the content?
4. Check shadow consistency — if used, shadows should share the same color, angle, and spread logic.
5. Look for any element that appears "off" and ask why.
6. Add one deliberate "moment" — a display treatment, pull quote, bold typographic choice that shows intent.
7. Remove anything decorative that isn't working for the composition.

**Checkpoint:** Is there a detail on this page that a careful observer would notice and appreciate?

---

## Phase 6: Reference Review

**Goal:** Validate the output against proven exemplars.

Questions:
- Does this have the informational clarity of Stripe?
- Does this have the restraint of Linear?
- Would a senior designer at Apple consider this premium?
- Does it pass Tufte's data-ink test on any charts?
- Is it as little design as possible (Rams)?

**Actions:**
- If something feels generic, go back to the technique catalog and apply one specific move.
- If something feels over-designed, remove the most recent addition.
- Run through the interrogation checklist for any remaining issues.

**Checkpoint:** Does this look like it was made by someone who cares deeply about design, or like a template that was filled in?

---

## When to Stop

Stop when:
- The interrogation checklist passes completely
- You can't identify a single thing you'd change if given more time
- The piece has at least one moment of genuine craft

Don't stop because:
- "It's good enough"
- "The user probably won't notice"
- "That's a minor detail"

The user might not articulate what they're seeing, but they'll feel the difference.
